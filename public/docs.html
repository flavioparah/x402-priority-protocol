x402-shield · developer guide

Get started in
under 5 minutes

x402-shield is an HTTP reverse proxy that turns any Solana RPC node into a priority-gated service. Under load it issues HTTP 402 Payment Required — agents that pay move to the front.

Quick test Deploy as operator Integrate SDK

How the protocol works

1
Agent sends
RPC request
2
Shield checks
load & budget
3
HTTP 402 +
signed nonce
4
Agent signs &
retries
5
RPC call
proxied through

No API keys. No IP whitelists. Identity = Ed25519 keypair. Payment = off-chain escrow debit. Overhead ≈ 8ms p95.

Quick test

Choose the fastest path to see the protocol in action.

Easy No install required. Hit the live demo endpoint from any terminal.
Step 1 — send an unauthenticated request (expect 402)
curl -s -o /dev/null -w "%{http_code}" \
  -X POST https://devnet.rpcpriority.com/rpc \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"getHealth","params":[]}'
# → 402
Step 2 — inspect the challenge payload
curl -s \
  -X POST https://devnet.rpcpriority.com/rpc \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"getHealth","params":[]}' | jq .
# → { "error": { "code": 402, "data": { "nonce": "...", "amount_micro_lamports": 1000, ... } } }
Note: devnet.rpcpriority.com runs with RPC_LOAD_FORCE=0.9 — 402 fires on ~90% of requests so you can see the challenge without waiting for real load. Want to bypass with a signed payment? See the SDK tab.

Implementation guide

Select your role for a step-by-step integration checklist.

1
Prerequisites

You need a Solana wallet (to receive payments) and a VPS with at least 2 vCPU / 4 GB RAM. Docker is recommended for production.

# Requirements
Node.js >= 22          # runtime
Redis >= 6             # atomic nonce store (optional for single-instance)
Docker + Docker Compose # recommended for deploy

# Get a Solana wallet address (if you don't have one)
solana-keygen new --outfile wallet.json
solana address -k wallet.json
2
Clone & configure

Copy .env.example to .env and fill the required values.

git clone https://github.com/flavioparah/x402-priority-protocol
cd x402-priority-protocol
cp .env.example .env
# .env — minimum required settings
PAYMENT_DESTINATION=YourSolanaWalletHere      # receives priority fees
REAL_RPC_URL=https://api.mainnet-beta.solana.com # upstream to proxy

# Gating — 402 fires when load exceeds this fraction (0–1)
RPC_LOAD_THRESHOLD=0.75
MAX_RPS=50                # what "100% load" means for your node

# Pricing in micro-lamports (Cenário 20× — production tier)
BASE_PRICE=20000          # min fee at threshold (= 20 lamports)
MAX_PRICE=1000000         # max fee at full load (= 1000 lamports)
3
Start & verify

Start the shield and confirm it's healthy. Set RPC_LOAD_THRESHOLD=0 to force 402 on every request during testing.

npm install
RPC_LOAD_THRESHOLD=0 npm start   # forces 402 on every request

# In another terminal:
curl http://localhost:3000/health | jq .
# → { "status": "ok", "load": 0.0, "rps": 0, "nonces_active": 0 }

curl -s -X POST http://localhost:3000/rpc \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"getHealth","params":[]}'
# → HTTP 402 with signed challenge
4
Run the test suite

43/43 tests pass on a fresh clone. Run all of them to confirm your environment is correctly configured.

npm run build              # compile TypeScript SDK
npm test                  # smoke test (requires shield running)
npm run test:detection    # fraud signal tests (offline)
npm run test:atomic       # atomic nonce (in-memory)
npm run test:cooperative-qos # QoS spec compliance
5
Production deploy (Docker + Traefik)

The production compose file enables Redis persistence, mainnet upstream, and Traefik TLS. Tested on Hostinger KVM4 (4 vCPU / 16 GB).

# Mainnet (real SOL payments, no trusted deposits)
docker compose -f docker-compose.mainnet.yml up -d --build

# Check logs
docker compose logs -f shield

# Devnet for staging
docker compose -f docker-compose.devnet.yml up -d --build
Cooperative QoS: add QOS_MODE=cooperative to your env to forward X-Priority-Score headers to an upstream operator that implements the cooperative QoS spec.
1
Install dependencies

The SDK requires @solana/web3.js, tweetnacl (Ed25519 signing), and bs58 (base58 encoding).

npm install @solana/web3.js tweetnacl bs58

# Copy the SDK file into your project
curl -o x402-client-sdk.ts \
  https://raw.githubusercontent.com/flavioparah/x402-priority-protocol/main/x402-client-sdk.ts

# Add to tsconfig.json if needed:
# "esModuleInterop": true, "moduleResolution": "node"
2
Generate keypair & fund escrow

Your agent's identity is an Ed25519 keypair — no API key required. Pre-fund an off-chain escrow so the Shield can debit payments instantly.

import { Keypair } from '@solana/web3.js';

// Generate once; persist secretKey in your config
const keypair = Keypair.generate();
const pubkey = keypair.publicKey.toBase58();

// Option A: demo (no real SOL) — for testing only
await fetch('https://devnet.rpcpriority.com/escrow/deposit-trusted', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ pubkey, amount_micro_lamports: 200_000 }),
});

// Option B: mainnet (real SOL, verified on-chain)
// Send SOL to the Shield's PAYMENT_DESTINATION address, then:
await fetch('https://api.rpcpriority.com/escrow/deposit', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ pubkey, tx_signature: '5z3kY…' }),
});
3
Wrap Connection with X402Provider

X402Provider extends Connection and intercepts all RPC calls. The 402 handshake is invisible to your application code.

import { X402Provider } from './x402-client-sdk';

const conn = new X402Provider(
  'https://devnet.rpcpriority.com/rpc',  // or api.rpcpriority.com
  keypair,
  {
    priorityBudget: 50_000,    // max µL per request (refuse if over)
    onChallenge: (c) => {      // optional: inspect challenge before paying
      console.log(`Paying ${c.amount_micro_lamports} µL`);
      return true;            // return false to abort
    },
  }
);
4
Use normally — 402 is invisible

Every existing @solana/web3.js call works as-is. Under load, the SDK intercepts the 402, signs the nonce, and retries — your app code sees only the final result.

// No code change needed in your application layer
const slot = await conn.getSlot();
const balance = await conn.getBalance(somePubkey);
const blockhash = (await conn.getLatestBlockhash()).blockhash;
const sig = await conn.sendRawTransaction(rawTx);

// Check your escrow balance any time
const res = await fetch(
  `https://api.rpcpriority.com/escrow/balance/${pubkey}`
).then(r => r.json());
console.log(res.balance_micro_lamports, 'µL remaining');
Trust-Score tip: your discount grows automatically with every paid request. At score 100 you pay 50% less. Check your score at /explorer or via /reputation/:pubkey.

API quick reference

Endpoint Method Description
/rpc POST JSON-RPC proxy. Returns 402 under load or 200 on pass-through.
/escrow/deposit POST Verify an on-chain tx and credit escrow. Body: { pubkey, tx_signature }.
/escrow/deposit-trusted POST Demo-only deposit (no on-chain tx). Requires ESCROW_TRUST_DEPOSITS=1.
/escrow/balance/:pubkey GET Returns { balance_micro_lamports } for the given agent pubkey.
/reputation/:pubkey GET Trust-Score (0–100), paid count, discount %, history.
/health GET Shield health, current load fraction, active nonce count, rps.
/stats/leaderboard GET Top 10 agents by Trust-Score.
/stats/qos GET Queue depth, in-flight count, avg wait time, overload events.
