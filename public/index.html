O protocolo x402 usa HTTP 402 para criar um mercado de acesso prioritário a nós RPC na Solana. Agentes autônomos pagam micro-taxas e operam sem interrupção — mesmo sob congestionamento máximo.

Cinco etapas.
Invisível para
o agente.

A negociação acontece inteiramente na camada de transporte. O código do agente não precisa mudar — apenas trocar o endpoint.

01

Handshake cego

O agente envia uma requisição RPC normal. Não sabe que existe um Shield à frente.

POST /rpc {"method":"getAccountInfo"}
02

Interceptação 402

O Shield detecta congestionamento. Bloqueia com HTTP 402 + nonce único + preço dinâmico.

X-x402-Amount: 3200 · X-x402-Nonce: a1b2c3
03

Avaliação do orçamento

O SDK verifica se o preço cabe no priorityBudget. Acima do limite, lança X402BudgetExceededError.

04

Liquidação off-chain

Assina o payload com Ed25519. Reenvia a requisição com o token de autorização. Zero espera de bloco.

Authorization: x402 sig.pubkey.msg
05

Validação e passagem

Shield verifica assinatura + nonce + saldo no escrow. Válido → encaminha ao RPC real. Latência total: ~4ms.

200 OK — overhead: +4ms

Três peças.
Qualquer stack.

Cada componente pode rodar de forma independente. O Shield é agnóstico ao cliente — qualquer agente que entenda HTTP pode negociar.

[A]

x402-Shield

Proxy reverso posicionado à frente do nó RPC. Monitora backpressure, emite desafios 402 com preço dinâmico e valida provas de pagamento.

Node.js 18+ Express http-proxy-middleware Redis (prod)
[B]

x402-Client-SDK

Estende o Connection da Solana. Intercepta erros 402, negocia o pagamento via Ed25519 e refaz a requisição — tudo transparente para o agente.

TypeScript 5 @solana/web3.js tweetnacl bs58
[C]

Priority-Gateway

Cofre de saldo (escrow) para liquidação instantânea. MVP usa memória + Map. Produção usa programa Anchor on-chain ou canal de estado.

Map (MVP) Anchor (v2) State Channels Redis

Só saldo.

O x402 é um protocolo aberto e permissionless. Você não assina nada — conecta o SDK, deposita saldo no escrow e seus agentes operam com prioridade de forma autônoma. O custo é exatamente o que a rede cobra, no momento em que cobra.

01

Conectar o Shield

Aponte o endpoint do seu agente para o x402-Shield em vez do RPC direto. Uma linha de configuração.

"http://shield.x402.io/rpc"
→
02

Depositar saldo

Deposite SOL no escrow do Shield via carteira ou instrução on-chain. Sem mínimo obrigatório.

depositEscrow(shield, 1_000_000 µL)
→
03

Operar

O agente negocia e paga automaticamente. Você recebe telemetria. Quando o saldo zera, recarrega.

onPayment: ({amount}) => log(amount)
